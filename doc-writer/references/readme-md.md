# README.md Rules

**Template:** `assets/readme-template.md` — start here for new READMEs. Fill in the `{{ placeholders }}` and delete sections that don't apply.

**Audience:** Someone who has never seen the project — a new contributor, a future self, or a curious reader. They need orientation, not architecture.

## What belongs

- One-line pitch: what does this project do and for whom
- Prerequisites and setup steps that actually work (verify them if you can)
- How to run it in development and production
- The minimum a contributor needs to know to make a change and test it

## What doesn't belong

- Architecture diagrams or system design — that's the design doc
- Agent-specific rules or conventions — that's CLAUDE.md
- Exhaustive feature lists — link to the design doc instead
- Internal decisions or rationale — a new user doesn't need to know why

## Tone

Direct and welcoming. Assume competence but not familiarity. Every sentence should serve someone trying to get started.
